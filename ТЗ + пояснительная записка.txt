Задание (ТЗ)
Что делаем: Сайт «Weather Journey», который показывает погоду и запоминает запросы.
Зачем: Чтобы не вводить по сто раз одни и те же города.
Что должно быть:
Поле для поиска (тянет данные через API OpenWeatherMap).
Автоматическое сохранение истории в базу (SQLite + SQLAlchemy).
Список из 5 последних городов под формой.
Кнопка «Очистить», чтобы быстро стереть историю из базы.
Пояснительная записка
Как это работает:
Сделал сервис на Flask. Главная фишка — база данных (ORM), которая сама записывает каждый поиск. Теперь, если обновить страницу, история никуда не денется.
Техника:
Бэкенд: Flask. Управляет всем процессом и запросами.
База: SQLAlchemy. Хранит историю (название, градусы, описание). Это удобнее, чем обычный SQL.
API: Данные залетают в формате JSON прямо с сервера погоды.
Визуал: Использовал Bootstrap 5 и немного своего CSS. Сделал стильные карточки с эффектом стекла и градиент на фоне.
Итог:
Получился живой сайт с базой данных и рабочим API. Всё по требованиям, код чистый, дизайн адаптивный.
